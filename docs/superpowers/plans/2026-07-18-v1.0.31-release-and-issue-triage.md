# v1.0.31 Release and Issue Triage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the verified issue #111 and copy/download hardening as v1.0.31, then close only issues already proven fixed while leaving residual-quality research open.

**Architecture:** Keep the release scoped to the 13 commits already ahead of `origin/main`. Regenerate versioned userscript and extension artifacts from `package.json`, validate locally and in GitHub Actions, publish the tag and assets, then apply evidence-based issue updates.

**Tech Stack:** Node.js, pnpm, Git, GitHub CLI, repository release validation scripts.

## Global Constraints

- Use `pnpm` for every Node.js package-management command.
- Do not start a local development server; reuse the user's existing `pnpm dev` process only for manual browser validation.
- Do not claim issue #101 or #103 is fully fixed.
- Preserve release scope: issue #111, request-layer copy/download hardening, and the real-page regression probe.
- Publish SDK/package only because the shipped request-layer and core changes are package-facing.
